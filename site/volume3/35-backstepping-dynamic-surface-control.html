第35章 反步控制与动态面控制

35.1 问题背景

反馈线性化和 NDI 侧重把输入输出关系直接反解。反步控制则更适合具有串级或严格反馈结构的系统：先设计虚拟控制，再逐层递推到真实输入。高超声速纵向控制中的“高度 -> 航迹角 -> 攻角 -> 俯仰角速度 -> 舵面”链条，正是理解反步法的典型入口。

反步法的价值，在于它能把层级关系一点点展开。它尤其适合那些能够自然拆成若干层虚拟控制的飞行控制问题。

35.2 严格反馈结构

典型二阶 strict-feedback 系统可写为：

ẋ1=f1(x1)+g1(x1)x2,ẋ2=f2(x1,x2)+g2(x1,x2)u \dot x_1 = f_1(x_1) + g_1(x_1)x_2,\qquad \dot x_2 = f_2(x_1,x_2) + g_2(x_1,x_2)u

其中，x1x_1 是上层状态，x2x_2 是下层状态，uu 是真实输入。 其中，f1(x1)f_1(x_1) 是上层漂移项，g1(x1)g_1(x_1) 是上层输入增益函数，f2(x1,x2)f_2(x_1,x_2) 是下层漂移项，g2(x1,x2)g_2(x_1,x_2) 是下层输入增益函数。

目标是让 x1x_1 跟踪参考 x1cx_{1c}。反步思想是先把 x2x_2 当成虚拟控制，设计期望 x2cx_{2c}，再设计真实输入 uu 让 x2x_2 跟踪 x2cx_{2c}。

35.3 一阶虚拟控制

定义误差：

e1=x1−x1c e_1 = x_1 - x_{1c}

其中，e1e_1 是第一层误差，x1cx_{1c} 是参考。 其中，x1x_1 是上层状态，x1cx_{1c} 是上层参考命令。

则可令：

x2c=−k1e1 x_{2c} = -k_1 e_1

其中，k1k_1 是第一层反馈增益。

这一步只是把 x2x_2 当成可直接指定的虚拟控制。下一步必须让真实 x2x_2 跟踪 x2cx_{2c}。

35.4 第二层误差

定义：

e2=x2−x2c e_2 = x_2 - x_{2c}

其中，e2e_2 是第二层误差。 其中，x2x_2 是下层状态，x2cx_{2c} 是虚拟控制命令。

设计 uu 时要使：

ė2=−k2e2 \dot e_2 = -k_2 e_2

其中，k2k_2 是第二层反馈增益。

那么可设计真实输入：

u=g2(x1,x2)−1(−f2(x1,x2)+ẋ2c−k2e2) u = g_2(x_1,x_2)^{-1}\left(-f_2(x_1,x_2) + \dot x_{2c} - k_2 e_2\right)

其中，g2(x1,x2)g_2(x_1,x_2) 是输入增益函数。

35.5 Lyapunov 递推

取 Lyapunov 函数：

V=12e12+12e22 V = \frac{1}{2}e_1^2 + \frac{1}{2}e_2^2

其中，VV 是候选 Lyapunov 函数。

若增益选择合理，则可使：

V̇≤−k1e12−k2e22 \dot V \le -k_1 e_1^2 - k_2 e_2^2

这说明误差收敛方向是明确的，且每一层的设计都在为整体稳定性服务。

35.6 动态面控制

反步法的困难在于需要对虚拟控制反复求导。动态面控制通过加一阶滤波器来减轻这一负担：

τfη̇+η=x2c \tau_f \dot \eta + \eta = x_{2c}

其中，η\eta 是滤波后的命令，τf\tau_f 是滤波时间常数。

然后用 η\eta 代替直接求导后的虚拟控制。这样做的核心目的是避免高阶求导带来的噪声放大和符号复杂度。

35.7 命令滤波误差

滤波会引入误差：

η̃=η−x2c \tilde \eta = \eta - x_{2c}

其中，η̃\tilde \eta 是滤波误差。

稳定性证明必须说明 η̃\tilde \eta 如何进入闭环，并证明它有界或被补偿。若只加滤波器却不分析滤波误差，只能称为带命令滤波的工程实现，不能称为完整命令滤波反步证明。

35.8 饱和和辅助系统

反步控制律通常先得到未饱和命令：

u0=g2(x1,x2)−1(−f2(x1,x2)+η̇−k2e2) u_0 = g_2(x_1,x_2)^{-1}\left(-f_2(x_1,x_2) + \dot \eta - k_2 e_2\right)

其中，u0u_0 是前馈或补偿项。 其中，f2(x1,x2)f_2(x_1,x_2) 是下层漂移项，g2(x1,x2)g_2(x_1,x_2) 是下层输入增益函数，η\eta 是滤波后的虚拟控制命令。

若执行机构饱和，可定义饱和误差：

eu=usat−u0 e_u = u_{\mathrm{sat}} - u_0

其中，usatu_{\mathrm{sat}} 是饱和后的输入。 其中，u0u_0 是未饱和输入命令，usatu_{\mathrm{sat}} 是执行机构饱和后的实际输入，eue_u 是饱和误差。

可设计辅助系统来吸收饱和误差，防止闭环发散。

35.9 与约束控制的连接

反步法可以和 BLF、预设性能或 CBF 结合。例如，若高度约束写成：

hmin⁡≤h≤hmax⁡ h_{\min} \le h \le h_{\max}

则可以把约束映射进误差变量或安全函数中，使控制律在设计阶段就考虑边界。 其中，hh 是高度状态，hmin⁡h_{\min} 和 hmax⁡h_{\max} 分别是高度下界和上界。

若外层高度命令已经超出当前速度和航迹能力，就应先做命令可行化、限幅或降级，而不是强行输出不可实现的虚拟控制。

35.10 高超声速应用边界

反步法适合严格反馈结构明显的任务段、需要分层稳定证明的场景、与约束函数结合的轨迹跟踪，以及需要显式处理虚拟控制的纵向内外环。

它的主要风险是虚拟控制求导复杂、命令滤波引入延迟、执行机构饱和时需要辅助系统、对模型耦合和符号推导要求高。

35.11 本章小结

反步控制和动态面控制把串级系统的层级关系写成了递推稳定设计。它们适合高超声速飞行器的分层纵向控制，但必须和命令滤波、饱和处理以及约束接口一起使用，才能真正落到工程闭环里。
