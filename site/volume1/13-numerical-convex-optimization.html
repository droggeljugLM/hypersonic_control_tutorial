第13章 数值优化与凸优化基础

本页是章节正文页。左侧目录用于回到本卷其他章节，底部导航用于前后章跳转。

13.1 问题背景

高超声速控制中的很多问题最终会落到优化：

  1. 轨迹优化；
  2. 模型预测控制；
  3. CLF-CBF-QP；
  4. 控制分配；
  5. 鲁棒控制权重选择；
  6. 参数辨识；
  7. 故障后重构。

因此，参考书级教程必须讲数值优化和凸优化基础。否则读者看到 MPC、序列凸优化或安全滤波时，只会把求解器当黑箱。

一般优化问题可写为

min⁡zf(z)s.t.gi(z)≤0,i=1,…,m,hj(z)=0,j=1,…,p. \min_{z}\; f(z) \quad\text{s.t.}\quad g_i(z)\le 0,\ i=1,\ldots,m,\qquad h_j(z)=0,\ j=1,\ldots,p.

其中 zz 是决策变量，f(z)f(z) 是目标函数，gi(z)≤0g_i(z)\le 0 是不等式约束，hj(z)=0h_j(z)=0 是等式约束。

在高超声速控制中，zz 不一定只是飞行器状态，也可能是控制输入序列、轨迹节点、舵面分配量、滤波器参数、鲁棒权重或故障重构变量。因此写优化问题时必须先说明决策变量。

13.2 优化问题的基本组成

一个可复核的优化问题至少包含：

组成 需要说明的问题
决策变量 优化器真正选择什么
目标函数 最小化跟踪误差、能量、残差、时间还是风险
等式约束 动力学、运动学、终端条件、分配等式
不等式约束 路径、输入、输入速率、安全裕度
参数 状态、参考、气动表、质量、环境
求解方法 QP、SOCP、NLP、SCP、枚举或启发式
失败处理 不可行、超时、数值病态时如何处理

若只写“求解一个优化问题”，但不说明这些组成，就无法判断控制器是否可实现，也无法复现实验。

13.3 无约束优化

无约束优化的一阶必要条件为

∇f(z⋆)=0, \nabla f(z^\star)=0,

其中 z⋆z^\star 是最优解，∇f(z⋆)\nabla f(z^\star) 是目标函数的一阶梯度。

如果还考察二阶条件，则 Hessian 矩阵

∇2f(z⋆) \nabla^2 f(z^\star)

其中 ∇2f(z⋆)\nabla^2 f(z^\star) 是目标函数的 Hessian 矩阵。它应当在最优点附近满足适当的正定性或半正定性要求。

在控制中，无约束优化可用于参数拟合、权重整定和最小二乘问题。但真实飞行问题通常有大量约束，因此无约束优化更多是基础工具。

13.4 KKT 条件

对约束优化问题，引入拉格朗日函数

ℒ(z,λ,μ)=f(z)+λ⊤h(z)+μ⊤g(z), \mathcal{L}(z,\lambda,\mu)=f(z)+\lambda^\top h(z)+\mu^\top g(z),

其中 ℒ\mathcal{L} 是拉格朗日函数，λ\lambda 是等式约束乘子，μ\mu 是不等式约束乘子。

KKT 条件通常包括：

  1. 驻点条件；
  2. 原始可行；
  3. 对偶可行；
  4. 互补松弛。

这些条件解释了约束何时活跃。比如路径约束活跃时，对应乘子不为零，最优轨迹会沿安全边界附近飞行。

KKT 条件通常是必要条件。若问题是凸优化，并满足适当约束资格条件，KKT 条件也可成为全局最优的充分条件。

13.5 凸集和凸函数

集合 𝒞\mathcal{C} 凸，当且仅当对任意 x,y∈𝒞x,y\in\mathcal{C} 和 θ∈[0,1]\theta\in[0,1]，

θx+(1−θ)y∈𝒞. \theta x+(1-\theta)y\in\mathcal{C}.

其中 𝒞\mathcal{C} 表示凸集，θ\theta 是两点之间的插值权重。

函数 ff 凸，当且仅当

f(θx+(1−θ)y)≤θf(x)+(1−θ)f(y). f(\theta x+(1-\theta)y)\le \theta f(x)+(1-\theta)f(y).

其中 ff 是凸函数，θ\theta 仍然是插值权重。

凸优化的优势是局部最优即全局最优，数值求解更可靠。高超声速轨迹优化常需要把非凸问题转化或近似为一系列凸问题。

但“看起来像二次型”不一定就是凸问题。若 Hessian 不半正定，或约束集合非凸，求解器可能只能得到局部解。

13.6 二次规划

QP 形式为

min⁡x12x⊤Hx+q⊤xs.t.Ax≤b,Aeqx=beq. \min_{x}\ \frac{1}{2}x^\top Hx+q^\top x \quad\text{s.t.}\quad Ax\le b,\quad A_{\mathrm{eq}}x=b_{\mathrm{eq}}.

其中 xx 是决策向量，HH 是二次项矩阵，qq 是线性项，AA 和 AeqA_{\mathrm{eq}} 是约束矩阵，bb 和 beqb_{\mathrm{eq}} 是对应右端项。

如果 H≽0H\succeq 0，且约束是线性的，则 QP 是凸优化问题。高超声速控制里，CLF-CBF-QP、控制分配和部分 MPC 线性化子问题都常写成 QP。

13.7 SOCP 与锥约束

二阶锥规划适合表示范数约束。典型形式为

min⁡xc⊤xs.t.∥Aix+bi∥2≤ci⊤x+di. \min_x\ c^\top x \quad\text{s.t.}\quad \|A_i x+b_i\|_2\le c_i^\top x+d_i.

其中 c⊤xc^\top x 是线性目标，二范数约束可写成二阶锥形式。

在控制中，SOCP 常用于范数限制、鲁棒优化和某些分配问题。它比一般 NLP 更稳健，也比简单 QP 更灵活。

13.8 非线性规划

NLP 的一般形式是

min⁡zf(z)s.t.gi(z)≤0,hj(z)=0, \min_z f(z) \quad\text{s.t.}\quad g_i(z)\le 0,\ h_j(z)=0,

其中 ff、gig_i、hjh_j 可以非线性、非凸。高超声速轨迹优化、气动耦合控制和非线性 MPC 常属于 NLP。

NLP 的风险包括：

  1. 局部极小；
  2. 初始猜测敏感；
  3. 约束资格条件失效；
  4. 求解时间不可预测；
  5. 不可行时难以诊断；
  6. 强非线性导致线性化误差大。

因此，若使用非线性规划或非线性预测控制，应报告初始猜测、迭代次数、收敛容差、求解时间、不可行次数和失效回退逻辑。

13.9 序列凸优化

高超声速轨迹优化通常非凸。序列凸优化思想是从当前轨迹出发，线性化或凸化非凸项，求解一系列凸子问题。

典型步骤是：

  1. 选取当前轨迹；
  2. 在局部做线性化或凸化；
  3. 求解凸子问题；
  4. 更新轨迹；
  5. 重复直到收敛。

典型子问题可以写为

min⁡Δz12Δz⊤HΔz+c⊤Δzs.t.GΔz≤h. \min_{\Delta z}\ \frac{1}{2}\Delta z^\top H\Delta z+c^\top \Delta z \quad\text{s.t.}\quad G\Delta z\le h.

其中 Δz\Delta z 是当前迭代的增量变量，HH 是局部二次项矩阵，GΔz≤hG\Delta z\le h 是线性化后的约束。

13.10 变量尺度化与数值病态

高超声速优化常见问题之一是尺度差异太大。高度、速度、动压、热流、舵偏和终端误差可能量级完全不同。

若变量尺度不统一，求解器容易病态。一个简单的尺度化写法是

z̃=D−1z, \tilde{z}=D^{-1}z,

其中 zz 是原始变量，z̃\tilde{z} 是尺度化后的变量，DD 是对角尺度矩阵。

尺度化的目标不是让公式更好看，而是让优化问题更容易求、约束更容易比较、日志更容易解释。

13.11 MPC 与实时优化

MPC 每个采样时刻都解一个有限时域优化问题。若采样时刻为 kk，预测时域为 NN，则可写成

min⁡uk:k+N−1JNs.t.xi+1=F(xi,ui),i=k,…,k+N−1. \min_{u_{k:k+N-1}} J_N \quad\text{s.t.}\quad x_{i+1}=F(x_i,u_i),\ i=k,\ldots,k+N-1.

MPC 的优势是能显式处理约束。它的风险是求解时间、不可行和模型误差。因此，若使用 MPC 或非线性 MPC，应报告求解时间、递推可行性、不可行次数和失效回退逻辑。

13.12 控制分配 QP

若上层给出虚拟控制 νc\nu_c，执行机构偏角为 δ\delta，分配矩阵为 BaB_a，则控制分配可写成

min⁡δ12∥Baδ−νc∥W2s.t.δmin⁡≤δ≤δmax⁡,δ̇min⁡≤δ̇≤δ̇max⁡. \min_{\delta}\ \frac{1}{2}\|B_a\delta-\nu_c\|_W^2 \quad\text{s.t.}\quad \delta_{\min}\le \delta\le \delta_{\max},\ \dot{\delta}_{\min}\le \dot{\delta}\le \dot{\delta}_{\max}.

其中 δ\delta 是执行机构偏角，BaB_a 是分配矩阵，νc\nu_c 是虚拟控制，WW 是权重矩阵；∥⋅∥W\|\cdot\|_W 表示加权范数。

最容易被混淆的有三件事：

  1. 上层优化变量是虚拟控制，不是实际舵偏；
  2. 下层优化变量是执行机构偏角，它受到位置和速率限制；
  3. 即使上层 QP 可行，若下层分配残差很大，真实系统仍可能无法执行安全命令。

13.13 轨迹优化与 MPC 交付例题

对速度高度跟踪和动压走廊问题，轨迹优化可写成

min⁡x0:N,u0:N−1∑k=0N−1ℓ(xk,uk)+ϕ(xN)s.t.xk+1=F(xk,uk). \min_{x_{0:N},u_{0:N-1}}\sum_{k=0}^{N-1}\ell(x_k,u_k)+\phi(x_N) \quad\text{s.t.}\quad x_{k+1}=F(x_k,u_k).

交付时必须说明：

  1. FF 是真实非线性模型、线性化模型、LPV 模型还是学习模型；
  2. 动压约束在预测时域内如何计算；
  3. 若 q(xk)≤qmax⁡q(x_k)\le q_{\max} 不可行，是放松参考、放松终端，还是使用回退；
  4. 是否检查节点之间的动压峰值；
  5. 求解时间是否小于采样周期；
  6. 多初值和扰动下不可行次数是多少。

13.14 优化证据层次及其与后续章节的关系

优化方法的证据强度可以按下表分层。

等级 证据 能说明什么 不能说明什么
O0 只写出优化公式或调用求解器 问题有形式化表达 不能说明可解、可复现或可实时
O1 给单场景解、目标值和约束值 该场景可能求解成功 不能说明鲁棒性和实时性
O2 给求解器日志、不可行次数、求解时间和约束违反 数值实现可复核 不能证明所有任务段可靠
O3 给多初值、多飞行点、扰动、故障和采样周期扫描 教学级闭环证据较完整 不能替代高保真模型和硬实时测试
O4 给跨工具复核、失败样本、失效回退效果和论文表格链路 可支撑论文或工程报告中的优化结论 仍需限定模型、飞行包线和求解器条件

13.15 优化对象包

一个优化问题能否进入后续章节，不能只看目标函数写得是否漂亮，而要看变量、约束、数值尺度和失败处理是否交代完整。建议把每个优化问题整理成下面的交付包。

项目 必须写清的内容 控制含义
决策变量 状态序列、输入序列、舵面量、松弛变量还是参数 决定优化器真正改变什么
参数包 状态、参考、气动表、质量、约束阈值和上一时刻输入 决定在线问题和离线问题的区别
目标函数 跟踪误差、控制能量、终端误差、风险项或参考修改代价 说明“最优”到底在优化什么
硬约束 动力学、输入边界、路径约束、终端约束和安全边界 决定是否允许违反
软约束 可松弛的次要目标、舒适性约束或次要任务指标 决定哪些部分可以降级
数值尺度 状态、输入、约束和权重的尺度化方式 决定求解器条件数
求解配置 求解器、容差、迭代次数、热启动和超时阈值 决定实时可实现性
失败处理 不可行、超时、数值失败时的失效回退 决定闭环不会因为优化失败而失控
日志字段 状态、目标值、约束违反、求解时间、活跃约束和降级标志 决定结果能否复核

13.16 常见错误

  1. 写出优化问题，但不说明变量、约束和求解方法。
  2. 把非凸问题当凸问题处理。
  3. QP 加了安全约束，却没有报告不可行次数。
  4. MPC 不报告求解时间。
  5. 序列凸优化只给最终轨迹，不报告迭代和约束违反。
  6. 控制分配优化不包含舵偏和舵速限制。
  7. 变量没有尺度化，导致求解器病态。
  8. 只检查离散节点，不检查节点之间路径约束。
  9. 用松弛变量掩盖安全约束失败。
  10. 求解器失败时没有失效回退。
  11. 把局部 NLP 解当作全局最优。
  12. 不记录初始猜测、warm start 和收敛容差。
  13. 只报告平均求解时间，不报告最大值和高分位值。
  14. 控制分配 QP 有数学解，但没有检查舵面动态是否能跟上。

13.17 本章小结

数值优化和凸优化是现代高超声速制导与控制的重要工具。本教程要求读者理解优化问题的变量、目标、约束、凸性、KKT 条件、求解方法和可信性边界。

后续 MPC、控制分配、CBF-QP、轨迹优化和再入制导都应建立在本章基础上。一个可用的优化章节还必须交付变量、参数、硬约束、软约束、求解器设置、失败处理和日志字段，使优化方法可以被复现、诊断和验证。
