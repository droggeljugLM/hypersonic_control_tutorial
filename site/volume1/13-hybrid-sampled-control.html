混杂系统与采样控制专题

本页是章节正文页。左侧目录用于回到本卷其他章节，底部导航用于前后章跳转。

1.1 为什么高超声速控制需要混杂视角

高超声速飞行器控制不是纯连续时间问题。飞行器动力学常写成连续微分方程，但真实闭环包含数字计算机、采样传感器、离散控制律、执行机构限幅、模式切换、故障逻辑、安全滤波启停和任务阶段切换。

连续控制律推导之后，还必须回答一个工程控制问题：连续对象、离散控制器、模式逻辑和约束保护合在一起之后，闭环是否仍然稳定、安全、可实现。

设连续对象为

ẋ=fq(x,u), \dot{x}=f_q(x,u),

这里的 x 是状态，u 是控制输入，q 是离散模式。

其中 xx 是状态，uu 是控制输入，qq 是离散模式。若控制器在采样时刻 tk=kTst_k=kT_s 更新，则数字控制律可以写成

uk=κ(xk,qk), u_k=\kappa(x_k,q_k),

这里的 u_k 是第 k 个采样时刻的控制律输出，x_k 是采样状态，q_k 是采样模式，kappa 表示数字控制律。

其中 TsT_s 是采样周期，xk=x(tk)x_k=x(t_k)，qkq_k 是第 kk 个采样时刻的模式。

1.2 离散化、零阶保持与精确离散模型

若连续系统为

ẋ=Ax+Bu, \dot{x}=Ax+Bu,

这里的 A 是连续系统矩阵，B 是输入矩阵，x 是状态，u 是连续输入。

并采用零阶保持，则离散模型为

xk+1=Adxk+Bduk, x_{k+1}=A_d x_k+B_d u_k,

这里的 x_{k+1} 是下一采样时刻状态，A_d 是离散状态矩阵，B_d 是离散输入矩阵。

其中 Ad=eATsA_d=e^{AT_s}，而

Bd=∫0TseAτBdτ. B_d=\int_0^{T_s}e^{A\tau}B\,d\tau.

这里的积分给出了零阶保持下输入在一个采样周期内的等效作用。

这里 AA 是连续系统矩阵，BB 是输入矩阵，TsT_s 是采样周期。注意离散模型不是把 AA 简单乘以 TsT_s。当采样周期较大或系统模态很快时，粗糙离散化会改变稳定性判断。

1.3 离散时间稳定性

离散线性系统

xk+1=Adxk x_{k+1}=A_d x_k

这里的 A_d 是离散系统矩阵，x_k 是离散状态。

渐近稳定的一个常见条件是 AdA_d 的特征值都在单位圆内。

若存在离散 Lyapunov 函数

Vk=xk⊤Pxk, V_k=x_k^\top P x_k,

这里的 V_k 是离散 Lyapunov 函数值，P 是正定矩阵。

其中 P≻0P\succ 0，则若

Vk+1−Vk<0, V_{k+1}-V_k<0,

这里的差分表示 Lyapunov 函数在相邻采样时刻的下降量。

可证明闭环稳定。这里 VkV_k 是第 kk 个采样时刻的 Lyapunov 函数值。

对于混杂系统，单个模式稳定不代表任意切换都稳定。必须说明切换逻辑、驻留时间和模式间兼容性。

1.4 模式切换与守卫条件

设离散模式集合为

𝒬={q1,q2,…,qm}, \mathcal{Q}=\{q_1,q_2,\ldots,q_m\},

这里的 Q 表示离散模式集合，q_i 表示第 i 个模式。

其中每个 qiq_i 对应一个控制目标或约束配置。

模式切换可写成

qk+1=g(qk,xk), q_{k+1}=g(q_k,x_k),

这里的 g 是模式更新规则，q_k 是当前模式，x_k 是当前状态。

其中 g(⋅)g(\cdot) 是模式更新规则。若引入守卫条件 hg(x)h_g(x)，则常见写法是：

hg(xk)≥0⇒qk+1=qnormal, h_g(x_k)\ge 0 \Rightarrow q_{k+1}=q_{\mathrm{normal}},

这里的 h_g 是守卫函数，满足条件时切回正常模式。

hg(xk)<0⇒qk+1=qprotect. h_g(x_k)<0 \Rightarrow q_{k+1}=q_{\mathrm{protect}}.

这里的第二个条件表示进入保护模式。

其中 qnormalq_{\mathrm{normal}} 是正常模式，qprotectq_{\mathrm{protect}} 是保护模式。

至少应明确：

  1. 模式集合 𝒬\mathcal{Q}；
  2. 每个模式的控制目标和约束；
  3. 进入/退出阈值；
  4. 最小驻留时间或保持时间；
  5. 是否存在内部状态重置。

若没有这一步，模式逻辑通常只是口头描述，不足以支撑后续验证。

1.5 采样间安全与收缩

采样点上安全，不等于采样间安全。若安全函数为

h(x)≥0, h(x)\ge 0,

这里的 h(x) 是安全函数，非负表示在安全集内。

而其变化率在一个采样周期内满足上界

|ḣ(x)|≤Lh, |\dot{h}(x)|\le L_h,

这里的 L_h 是安全函数变化率上界。

则可在数字实现中用保守收缩

h(xk)≥LhTs h(x_k)\ge L_h T_s

这里的收缩把连续时间安全裕度换成采样点上的保守裕度。

来降低采样间越界风险。

这里 LhL_h 是安全函数变化率上界，TsT_s 是采样周期。若系统在一个采样周期内快速接近动压、热流或攻角边界，即使 h(xk)h(x_k) 和 h(xk+1)h(x_{k+1}) 都非负，中间仍可能越界。

实际验证中应同时记录：

指标 含义
hmin⁡,sampledh_{\min,\mathrm{sampled}} 采样点上的最小安全函数
hmin⁡,denseh_{\min,\mathrm{dense}} 更密集积分点上的最小安全函数
NviolationinterN_{\mathrm{violation}}^{\mathrm{inter}} 采样点之间越界次数
TupdateT_{\mathrm{update}} 安全滤波更新周期
tsolvet_{\mathrm{solve}} 安全滤波求解时间

这能防止把“离散点通过”误认为“连续飞行过程安全”。

1.6 多速率控制与命令保持

高超声速制导层、姿态层、执行机构层往往采用不同更新周期。设制导层周期为 TgT_g，姿态层周期为 TcT_c，则常见结构是：

Tg≥Tc. T_g\ge T_c.

这里的 T_g 是制导层周期，T_c 是姿态层周期。

一个最简实现是制导命令保持：

rc(t)=rg(tk),t∈[tk,tk+1), r_c(t)=r_g(t_k),\qquad t\in[t_k,t_{k+1}),

这里的 r_g 是制导命令，r_c 是保持后的姿态层命令。

其中 rgr_g 是制导命令，rcr_c 是姿态层接收的保持命令。

这里 Γ\Gamma 应至少包含：

  1. 命令坐标变换；
  2. 限幅与限速；
  3. 当前模式下的安全或故障降级逻辑；
  4. 输入效能变化导致的可执行命令截断。

若制导层生成命令 rgr_g，姿态层实际执行命令为 rcr_c，则接口误差定义为

er=rc−rg. e_r=r_c-r_g.

这里的 e_r 是多速率接口误差。

其中 ere_r 是多速率接口误差。

1.7 采样与执行机构接口

设 ucu_c 是数字控制器给出的保持命令，uau_a 是执行机构真实输出，则控制链可以写成

ua=𝒜(uc), u_a=\mathcal{A}(u_c),

这里的 u_c 是数字控制器命令，u_a 是执行机构真实输出，A 表示执行机构映射。

其中 𝒜(⋅)\mathcal{A}(\cdot) 表示执行机构映射。

对高超声速控制，离散命令到真实执行之间通常存在饱和、速率限制、死区、卡滞和延迟。因此，连续稳定性证明必须和离散实现一起写。

1.8 采样间验证协议

混杂与采样控制至少应报告：

  1. 采样点上的安全指标；
  2. 采样间的安全指标；
  3. 模式切换时刻；
  4. 最短驻留时间；
  5. 求解时间与超时统计；
  6. 真实执行量与命令残差。

若时间有限，至少也要交出下面这份最小包：

类别 最小内容
采样/延迟 TsT_s、延迟拍数、保持方式
模式逻辑 模式集合、守卫条件、驻留时间
命令链 原始命令、保护修正、真实执行量
安全指标 采样点越界、采样间越界、最小裕度
实时性 平均求解时间、最大求解时间、超时次数
结果证据 恢复时间、任务误差、失败样本

1.10 本专题小结

混杂系统与采样控制把连续理论推向机载实现。对高超声速飞行器，采样、保持、延迟、模式切换、故障逻辑和安全保护都会改变闭环。连续稳定性证明是起点，不是结束。

合格的参考书写作应同时说明连续控制律、离散实现、模式逻辑、执行机构约束、采样间安全和实时计算证据。
